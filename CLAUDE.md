## Project Overview

Guac-Party deploys HA Apache Guacamole on a Proxmox cluster via HashiCorp Nomad. The admin workstation only needs Docker — Packer, Terraform, and Nomad CLI all run in containers via wrapper scripts in `docker/`.

## Pipeline

The deployment is a linear pipeline with manual steps between stages:

1. **`bootstrap/bootstrap.sh`** — Interactive. Creates Proxmox automation user/role/token, SDN, SSH keys, generates `terraform.tfvars` and `.env`, then invokes Packer and Terraform.
2. **Packer** (`docker/packer.sh`) — Builds Ubuntu 24.04 VM template with Docker CE + Nomad pre-installed.
3. **Terraform** (`docker/terraform.sh`) — Clones one Nomad VM per Proxmox node. Cloud-init writes Nomad config and starts the cluster.
4. **`nomad/deploy.sh`** — Deploys jobs in order: Vault → Traefik → Authentik → Guacamole. Pauses for manual Vault init/unseal and Authentik OIDC setup.

## Key Commands

```bash
# Full bootstrap (interactive, runs everything)
./bootstrap/bootstrap.sh

# Docker-wrapped tool invocations
./docker/terraform.sh init
./docker/terraform.sh apply
./docker/packer.sh build /workspace/packer/ubuntu-nomad
./docker/nomad.sh job run nomad/jobs/vault.nomad.hcl
./docker/nomad.sh job status vault

# Deploy all Nomad services (requires NOMAD_ADDR)
export NOMAD_ADDR=http://<nomad-vm-ip>:4646
./nomad/deploy.sh
```

## Architecture

**Every Nomad node runs both server and client roles.** The cluster size is variable — `bootstrap_expect` and `retry_join` are generated dynamically from the node list.

**Service discovery**: Nomad native (no Consul). Traefik reads service tags from the Nomad API at `127.0.0.1:4646` and creates HTTP routers dynamically.

**Routing pattern** (all Nomad jobs follow this):
```hcl
service {
  provider = "nomad"
  tags = [
    "traefik.enable=true",
    "traefik.http.routers.<name>.rule=Host(`<name>.internal`)",
    "traefik.http.routers.<name>.tls=true",
  ]
}
```

**Secrets injection** (Vault → Nomad template blocks):
```hcl
vault { policies = ["<service>"] }
template {
  data = <<-EOF
    {{ with secret "secret/data/<service>/db" }}
    PASSWORD={{ .Data.data.password }}
    {{ end }}
  EOF
  destination = "secrets/app.env"
  env         = true
}
```

**Sidecar pattern**: PostgreSQL runs as `lifecycle { hook = "prestart"; sidecar = true }` in both Authentik and Guacamole job groups. Within a task group, tasks reference each other via `NOMAD_IP_<port_label>` and `NOMAD_PORT_<port_label>`.

**Persistence**: Host volumes defined in the Nomad agent config (`/opt/volumes/{vault,authentik-db,guacamole-db}`) and mounted into job tasks. Created during Packer template build.

## Important Patterns

- **Docker wrappers** (`docker/*.sh`): All load `.env`, mount project root as `/workspace`, pass through relevant env vars. `nomad.sh` uses `--net=host`.
- **Bootstrap lib functions**: Sourced by `bootstrap.sh`. The Proxmox API library (`proxmox-api.sh`) uses cookie-based auth via `pve_auth()` and exports `PVE_TICKET`/`PVE_CSRF` for subsequent calls.
- **Terraform modules**: `proxmox-sdn` creates the guest network (out of scope for services). `nomad-node` uses `for_each` over the node map — one VM per Proxmox host.
- **Cloud-init template chain**: `main.tf` → `cloud-init.yaml.tpl` (embeds `nomad.hcl.tpl` via nested `templatefile()`). The Nomad config is indented into the YAML `write_files` block.
- **Vault Raft discovery**: Vault instances find each other via `{{ range nomadService "vault" }}` in the Raft `retry_join` config template.
- **OIDC is opt-in**: The Guacamole OIDC template block is commented out. Enable it after configuring Authentik.
